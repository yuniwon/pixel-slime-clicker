# Pixel Slime Clicker Playtest Guide

목표: 자동 `launchCheck()`가 잡지 못하는 실제 재미, 이해도, 재방문 의향을 20분 안에 확인한다. 이 문서는 테스트 절차와 질문지, 로컬 텔레메트리 공유 방법을 정리한다.

## 테스터 안내 문구

아래 문구를 그대로 전달한다.

> 귀여운 슬라임 클리커를 20분 정도 편하게 플레이해 주세요. 정답 루트는 없고, 놓쳐도 손해인 행동은 거의 없습니다. 막히거나 헷갈리는 순간이 있으면 말로 중얼거려 주세요. 끝나면 짧은 질문 몇 개와 5점 설문에 답해 주세요.

주의:
- 개발자가 옆에서 힌트를 많이 주지 않는다.
- 테스터가 물어보면 "어디가 궁금했는지"를 먼저 기록한다.
- 버그나 막힘이 있어도 바로 고치지 말고 재현 메모를 남긴다.

## 20분 테스트 절차

### 준비 2분

1. 새 시크릿 창 또는 기존 세이브 초기화 상태로 시작한다.
2. 화면 녹화가 가능하면 켠다.
3. 관찰자는 시작 시간을 적고, 테스터에게 안내 문구만 읽어준다.

### 플레이 20분

관찰자는 개입하지 않고 아래 시점을 기록한다.

| 구간 | 기대 흐름 | 기록할 것 |
| --- | --- | --- |
| 0~1분 | 클릭, 코인, 퍼펙트 말랑, 첫 친구 | 첫 친구를 스스로 이해했는지 |
| 1~5분 | 첫 업그레이드, 친구 행동, 첫 소원, 첫 황금별 | 탭 이동이 자연스러운지, 별을 놓치지 않는지 |
| 5~10분 | 첫 진화, 첫 축복, 별의 기록 첫 해금 | 축복이 "이번 생의 성격"처럼 느껴지는지 |
| 10~20분 | 오늘의 의식, 젤리 결정, 황금 소원, 별가루 꾸미기, 도감/방 | 기능이 많아도 다음 행동이 보이는지 |

### 종료 3분

1. 설정에서 세이브 코드를 내보낸다.
2. 콘솔에서 `telemetryDump()`를 실행한다.
3. 아래 질문과 5점 설문을 진행한다.

## 관찰 포인트

- 첫 30초 안에 슬라임을 누르고 보상을 이해하는가
- 첫 친구 구매 버튼을 스스로 찾는가
- 친구가 자동으로 코인을 벌어준다는 사실을 알아차리는가
- 소원, 황금별, 먹구름, 젤리 결정이 "부담 없는 초대"처럼 보이는가
- 축복 선택이 단순 버프 카드가 아니라 이번 생의 방향처럼 느껴지는가
- 10분 이후 탭이 많아졌을 때 길을 잃는가
- 방, 도감, 별의 기록이 왜 있는지 감정적으로 이해되는가
- 모바일 또는 작은 화면에서 버튼/텍스트가 답답하지 않은가
- 게임을 끄기 전에 "조금만 더" 하고 싶은 목표가 남는가

## 플레이 후 질문

1. 처음 1분 동안 무엇을 해야 하는지 명확했나요?
2. 가장 먼저 재미있다고 느낀 순간은 언제였나요?
3. 가장 헷갈렸던 탭이나 문구는 무엇이었나요?
4. 친구, 소원, 황금별, 먹구름 중 가장 손이 갔던 것은 무엇인가요?
5. 첫 진화나 첫 축복이 기억에 남았나요?
6. 별가루와 꾸미기는 보상처럼 느껴졌나요, 아니면 나중 일처럼 느껴졌나요?
7. 20분 뒤에도 남은 목표가 보였나요?
8. 다시 한다면 다른 방식으로 플레이해 보고 싶은가요?

## 5점 설문

1점은 전혀 아니다, 5점은 매우 그렇다.

| 항목 | 질문 |
| --- | --- |
| 몰입 | 시간이 빨리 지나간다고 느꼈나요? |
| 명확함 | 다음에 무엇을 하면 좋을지 대체로 보였나요? |
| 손맛 | 클릭, 보상, 소리, 움직임이 기분 좋았나요? |
| 선택의 의미 | 친구/강화/축복 선택이 내 플레이에 영향을 준다고 느꼈나요? |
| 호기심 | 아직 못 본 기능이나 다음 모습이 궁금했나요? |
| 부담 없음 | 일일 요소나 이벤트가 숙제처럼 느껴지지 않았나요? |
| 재방문 의향 | 내일 다시 열어볼 마음이 있나요? |

마지막에 별도 질문:

> 또 할래?

답변은 `예 / 아니오 / 모르겠음` 중 하나로 받고, 이유를 한 문장으로 적는다.

## 텔레메트리 수집

게임은 외부로 데이터를 보내지 않는다. 플레이가 끝나면 개발자 콘솔에서 아래를 실행한다.

```js
telemetryDump()
```

요약에 포함되는 항목:

- 세션 수
- 총 플레이 시간
- 평균 세션과 최근 세션 길이
- 첫 친구까지 걸린 시간
- 첫 진화까지 걸린 시간
- 첫 축복 선택까지 걸린 시간
- 소원 완료 수
- 황금별 응답률
- 먹구름 참여율과 성공률
- 길 잃은 별똥이 구조율
- 도전 런 시작 수
- 리롤/스킵 수
- 우주 씨앗 조건 진행도
- 현재 레벨, 단계, 환생, 영혼, 별가루

기존 세이브에서 이미 지나간 첫 친구/첫 진화/첫 축복은 `기록 전 달성`으로 표시될 수 있다. 신규 세이브 플레이테스트에서는 시간으로 기록된다.

## 세이브 코드 공유 가이드

테스터에게 아래 순서로 안내한다.

1. 게임 오른쪽 위 설정 버튼을 누른다.
2. `세이브 코드 내보내기`를 누른다.
3. 복사된 긴 코드를 개발자에게 전달한다.
4. 가능하면 `telemetryDump()` 결과와 설문 답변도 함께 전달한다.

공유 받을 때 함께 적을 정보:

- 테스트 날짜
- 기기와 브라우저
- 화면 크기 또는 모바일/데스크톱 여부
- 플레이 시간
- 또 할래 답변
- 가장 헷갈린 순간 1개
- 가장 좋았던 순간 1개
