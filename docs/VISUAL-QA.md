# 시각 QA 프로토콜 — "AI로 만든 티" 잡아내기

배경: 2026-06-12 플레이테스트에서 액세서리 공중부양·왕관 충돌·별가루 스티커화를
스크린샷으로 보고도 통과시킴. 원인은 시력이 아니라 **절차** — 검증 편향(의도한 것이
있는지만 확인), 저해상도 캡처, 정지 1프레임, 미적 기준 부재.
이 문서는 모든 비주얼 변경에 강제하는 비판 패스 절차다.

## 절차 (비주얼 변경마다)

1. **풀해상도 합성** — 게임 캡처는 다운스케일로 디테일이 뭉개진다.
   에셋 차원의 배치는 .NET/캔버스 합성으로 원본 해상도에서 본다 (tools 패턴: tmp-fit-check.ps1).
2. **비판 패스 (의무)** — "의도대로 됐나?"가 아니라 **"이 그림에서 싸 보이는 것 5개를 찾아라"**
   를 명시적으로 자문하고 답한다. 0개면 더 찾는다. 확인 편향 차단이 목적.
3. **상태 매트릭스** — 한 상태가 아니라 조합 전체: 5단계 × 성향 × 프레임.
   한 칸이라도 깨지면 그 조합은 끄는 게 낫다 (어색한 노출 < 비노출).
4. **모션 확인** — "N프레임 효과"는 정지샷으로 판정 금지. 프레임 페어를 나란히 놓거나
   GIF로 본다. 정지로 어색하면 모션으로도 대개 어색하다.
5. **양 뷰포트 실런타임** — 합성 통과 후 헤드리스 크롬(데스크탑+390px)으로 실화면 재확인.

## "AI 티" 체크리스트 (비판 패스에서 묻는 것)

- [ ] **공중부양**: 얹힌 요소가 대상의 곡면/윤곽을 따라 닿아 있는가? 접촉 없는 모자·장식 = 스티커
- [ ] **요소 충돌**: 기존 디테일(왕관·표정·하이라이트)을 가리거나 어색하게 겹치는가?
- [ ] **그림자/접지**: 바닥에 놓인 것에 접지 그림자가 있는가? 없으면 "덩그러니"
- [ ] **스케일 정합**: 이웃 요소와 픽셀 밀도·외곽선 두께가 맞는가?
- [ ] **효과 vs 스티커**: 이펙트(반짝임·오라)가 대상과 상호작용하는가, 위에 붙어만 있는가?
- [ ] **배치 의도**: 그 자리에 있어야 할 이유가 읽히는가? (장식이 아무 데나 = 무의도)
- [ ] **팔레트 통일**: 새 에셋의 채도·명도 톤이 기존 톤과 맞는가?

## 합성 한계 주의

- pinkifyGray 등 런타임 보정은 .NET 합성에 적용되지 않는다 (회색 딸기 = 합성 아티팩트)
- CSS 필터·애니메이션은 합성에 없다 — 최종 판정은 실런타임

## 구조적 교훈

런타임 제네릭 오버레이(아무 단계에나 같은 액세서리)는 "붙인 느낌"을 구조적으로 못 벗어난다.
**품질 기준 = 에셋 차원에서 입혀 굽기** (단계별 변형 생성). 오버레이는 부유가 자연스러운
것(고리·말풍선)만 허용.
