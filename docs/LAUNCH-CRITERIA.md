# 출시 판정 기준 (Launch Readiness Criteria)

업계 표준(크래시프리율, 소프트런치 KPI 게이트, 플랫폼 인증 체크리스트, 성능 예산, 출시 사인오프)을
사전 출시 단계의 싱글플레이어 웹 게임에서 **지금 측정 가능한 형태**로 변환한 5가지 기준.
각 기준은 게임 내장 `launchCheck()` 가 자동 측정하며, 가중 평균 **99% 이상**이면 출시 가능 판정.

근거 출처: 크래시프리 세션율 99.5%+가 모바일 최저선이라는 업계 데이터, D1 40%/튜토리얼 완주
80~90% 소프트런치 게이트, 플랫폼 인증(TRC/lotcheck) 체크리스트 관행, 성능 예산·회귀 벤치마크 관행.

| # | 기준 | 업계 원형 | 가중치 | 측정 방법 |
|---|---|---|---|---|
| 1 | **안정성** | 크래시프리 세션율 ≥99.5% | 30% | 전 시스템 자동 플레이 시나리오 + 입력 퍼즈 중 콘솔 에러 0, NaN/Infinity 0, 세이브 무결성 |
| 2 | **온보딩 명료성** | 튜토리얼 완주율 80~90% 게이트 | 15% | 신규 유저 시뮬: 60초 내 첫 구매 도달, 모든 핵심 시스템에 힌트/팁 존재 |
| 3 | **콘텐츠 런웨이·페이싱** | 소프트런치 리텐션 게이트(D1/D7)의 사전 프록시 | 25% | simRun: 첫 환생 20~50분, 갈증 곡선(레벨 간격 증가), 첫 런 강화 소진 ≤60%, 장기 목표 잔존, 인플레 상한 |
| 4 | **성능** | 성능 예산(프레임/로드/메모리) | 15% | econTick 평균 <1.5ms, 핫패스 1만회 <300ms, 파티클/DOM 누수 0, 노드 수 상한 |
| 5 | **패키징·플랫폼 준수** | 플랫폼 인증(TRC) 체크리스트 | 15% | PWA manifest/아이콘/SW-캐시 정합, 세이브 코드 왕복, v1~v9 마이그레이션 전부 로드, 모바일 375px 오버플로 0 |

## 실행 방법

게임 콘솔에서 `launchCheck()` → `{ score, byCriterion, failures }`.
score ≥ 99 = 출시 가능. failures는 항목별 수정 지점.

## 기준 6: 재미 (사람 측정 트랙 — 자동 점수와 별개)

재미는 자동 측정 불가. 업계 방식(관찰 플레이테스트 / PXI류 설문 / 행동 텔레메트리)을 적용:

1. **5축 설문** (빌드마다 메인 테스터가 1~5점): 몰입 / 도전의 적절함 / 선택의 의미 / 호기심 / 재방문 의향.
   게이트: 평균 ≥ 4.0
2. **로컬 텔레메트리** (`telemetryDump()` 내장): 세션 수·평균 세션 길이·황금별 응답률·먹구름 참여율·리롤/스킵·도전 시작 수.
   외부 전송 없음, 세이브에 포함되어 세이브 코드로 공유 가능
3. **출시 전 외부 게이트**: 테스터 5~10명, 중위 플레이타임 ≥ 20분 + "또 할래?" ≥ 70%

## 자동 측정 불가 항목 (수동 체크리스트 — 점수 외)

- [ ] 게임위 자체등급분류 (토스/스토어 입점 직전)
- [ ] 개인정보처리방침 페이지 (저장은 로컬뿐이므로 간단)
- [ ] 스토어 스크린샷/설명 텍스트
- [ ] BGM (선택 — 현재 효과음만)
- [ ] 실기기 모바일 터치 감각 (시뮬 불가)
