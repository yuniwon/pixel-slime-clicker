너는 pixel-slime-clicker의 비주얼 디렉션/프론트엔드 정리 담당 Codex 에이전트다.

레포:
https://github.com/yuniwon/pixel-slime-clicker

중요:
내가 함께 첨부하는 레퍼런스 이미지는 “그대로 똑같이 복제하라”는 뜻이 아니다.
이 이미지는 게임의 최종 시각 방향을 잡기 위한 Visual North Star다.
현재 코드에 이미 많은 시스템이 들어오면서 이미지 구성과 배치가 중구난방이 되었으므로, 이번 작업의 목표는 새 기능 추가가 아니라 “작은 말랑 우주 극장”이라는 하나의 기준으로 메인 화면과 탭 UI를 정돈하는 것이다.

절대 하지 말 것:
- 경제 수치 변경 금지
- 새 게임 시스템 추가 금지
- 새 재화 추가 금지
- 대규모 로직 리팩터링 금지
- 저장 구조 변경 금지. 꼭 필요할 때만 SAVE_VERSION/migrate 처리
- 레퍼런스 이미지를 배경 이미지로 통째로 넣는 방식 금지
- 모든 요소를 항상 레퍼런스 이미지처럼 가득 보이게 만들기 금지
- UI를 모바일 광고 게임처럼 과하게 반짝이게 만들기 금지

이 이미지들의 공통 방향:
- 작은 말랑 우주 극장
- 깊은 보라 밤하늘
- 중앙 슬라임이 가장 큰 초점
- 친구들은 무대의 주민
- 장식은 무대 세트
- 하늘 요소와 바닥 요소가 분명히 구분됨
- 오른쪽/하단 패널은 어두운 보라색 픽셀 UI
- UI는 많지만 질서 있게
- 바닥 오브젝트는 접지 그림자
- 하늘 오브젝트는 은은한 글로우
- 전체는 풍성하지만 산만하지 않음

이번 작업의 큰 목표:
1. 레퍼런스 이미지를 기준으로 시각 규칙 문서화
2. 현재 메인 화면 레이어 구조를 명확히 목록화
3. 데스크톱/모바일 무대 좌표계를 도입하거나 기존 좌표를 중앙 관리 구조로 정리
4. 친구/장식/하늘 손님/소원 스트립/뉴스/상점 패널의 배치를 레퍼런스 방향으로 정렬
5. 탭별 패널 UI의 카드 문법을 통일
6. VISUAL-QA 기준으로 “AI로 만든 티”, 공중부양, 스티커 느낌, 접지 부재, 스케일 불일치를 줄임
7. launchCheck(), uxCheck()를 유지

# Visual Direction

## Core Fantasy
작은 말랑 우주 극장

## Target Player Feeling
- 귀여운 슬라임이 사는 작은 우주를 돌본다
- 친구들이 숫자 생산자가 아니라 방의 주민처럼 보인다
- 오늘 할 일이 있지만 숙제처럼 느껴지지 않는다
- 환생과 우주 씨앗은 반복의 흔적을 남긴다

## Visual Hierarchy
1. Slime
2. Coin / Level / Evolution Progress
3. Friends and Decor
4. Wish Strip / Daily Ritual
5. Shop Panel
6. Ambient Guests and Effects

## Stage Zones
- Sky Zone
  - 우주 고래
  - 코인 행성
  - 별의 정령
  - 황금별
  - 별똥이
  - 우주 창문
  - 우주 씨앗/별나무 상단
- Slime Zone
  - 슬라임
  - 콤보 오라
  - 성향 오라
  - 말풍선
- Ground Zone
  - 병아리
  - 고양이
  - 강아지
  - 아기 드래곤
  - 시간 거북
  - 쿠션
  - 연못
  - 매트
  - 램프
  - 코인 분수
  - 젤리 결정 화단
  - 코인 더미
- UI Zone
  - 뉴스
  - 레벨/코인/CPS
  - 오늘의 말랑
  - 소원 스트립
  - 상점 패널

## Layer Order
0 background stars
1 sky ambient
2 back decor
3 back friends
4 combo aura
5 slime
6 front friends
7 ground props
8 wish strip
9 event guests
10 toast/modal

## Density Rules
Early game:
- visible friends: 0~2
- visible decor: 0~1
- no cosmic tree
- no coin pile unless threshold reached

Mid game:
- visible friends: 3~5
- visible decor: 2~4
- wish strip visible
- one ambient sky object allowed

Late game:
- visible friends: 6~8
- visible decor: 4~6
- cosmic seed/tree allowed
- coin pile and crystal garden visible

Never:
- all 10 friends at full size all the time
- every decoration equally bright
- UI elements covering slime face
- unanchored ground props floating

## Size Rules
Desktop:
- slime: 280~340px visual height
- ground friend: 40~64px
- sky friend: 56~100px
- decor: 50~95px
- shop icon: 44~64px
- wish strip height: compact, no taller than necessary

Mobile:
- top stage: about 42~45dvh
- slime: tap-friendly but not crowding
- visible friends: 3~5 max
- visible decor: 2~4 max
- shop cards: 2~3 visible
- bottom nav/tabs touch-friendly

## Shadow and Glow Rules
- 바닥 오브젝트는 반드시 접지 그림자
- 하늘 오브젝트는 접지 그림자 없이 글로우
- UI는 drop-shadow만 사용
- 이펙트는 슬라임/오브젝트와 상호작용해야 하며 스티커처럼 붙으면 안 됨

## Color Palette
- deep violet
- lavender
- warm gold
- mint green
- strawberry pink
- soft teal
- avoid harsh neon
- avoid pure white blocks except tiny spark highlights

## Reference Image Interpretation
- 레퍼런스 이미지는 late-game / polished state 기준
- 실제 early-game 화면은 훨씬 덜 복잡해야 함
- 레퍼런스의 분위기, 위계, 색감, 패널 문법을 따르고 모든 요소를 1:1 복제하지 않는다

## Visual QA Checklist
docs/VISUAL-QA.md의 체크리스트를 요약해서 포함:
- 공중부양
- 요소 충돌
- 접지 그림자
- 스케일 정합
- 효과 vs 스티커
- 배치 의도
- 팔레트 통일

5단계. 메인 화면 실제 정렬
────────────────────────

현재 메인 화면을 레퍼런스 이미지 방향으로 정리한다.

수정 대상:
- renderDecor()
- rebuildFriends()
- crystal-bed position
- coin-pile position
- cosmic-seed-tree position
- news-ticker density
- wish-strip positioning
- mobile CSS

요구:
1. 슬라임이 항상 가장 큰 초점이어야 함.
2. ground 요소는 접지 그림자를 가져야 함.
3. sky 요소는 glow만 사용.
4. 친구는 레퍼런스처럼 반원형으로 배치하되 초반/중반/후반 밀도 제한.
5. 모바일에서는 친구 종별 최대 1마리, 전체 5마리 내외.
6. 우주 씨앗/별나무는 중앙 상징이지만 슬라임보다 강하면 안 됨.
7. 코인 더미/젤리 결정/소원 스트립이 서로 겹치면 안 됨.
8. 뉴스/토스트/버프 배너가 제목/레벨/코인을 가리면 안 됨.

6단계. 탭 UI 카드 문법 통일
────────────────────────

레퍼런스 이미지의 오른쪽 패널을 기준으로 탭별 카드 문법을 정리한다.
새 기능 추가 금지. 기존 정보를 더 보기 좋게 묶는 작업만 허용.

탭별 목표:

친구들:
- 주민 고용과 시너지 확인
- 카드 구성:
  icon / name / CPS / owned / payback / next synergy / cost / x1 x10 MAX
- 너무 긴 flavor text는 모바일에서 숨김

강화:
- 손맛 성장
- 카드 구성:
  icon / name / effect / level or purchased state / cost
- 업그레이드 아이콘과 cost 버튼 위치 통일

소원:
- 오늘의 작은 목표
- 황금 소원, 일반 소원 3개, 젤리 결정, 별가루 꾸미기 구분
- 목표 카드에는 progress bar가 명확해야 함

방:
- 흔적과 수집
- 장식 그리드, 도감, 별의 기록, 앨범, 우주 씨앗 상태 카드
- 장식은 레퍼런스처럼 grid 카드 문법으로
- 도감/기록은 너무 빽빽하면 접힘/섹션화 고려

환생:
- 더 깊어지는 선택
- 보유 영혼, 총 획득 영혼, 소프트캡 보너스, 환생 버튼, 특성 카드, 도전 런
- 숫자 표기는 실제 soulMul 기준 유지

설정:
- 세이브/사운드/진동/통계/개인정보/옵트아웃
- 위험 구역은 시각적으로 분리

모바일:
- 탭 버튼은 크게
- 카드 2~3개가 한 화면에 보이게
- x1/x10/MAX는 엄지로 누르기 쉬워야 함

- 친구와 장식이 무작위 배치가 아니라 의도된 주민/무대 세트처럼 보일 것
- 모바일에서 덜 복잡하고 실제로 터치하기 좋을 것
- 게임 로직과 경제가 바뀌지 않을 것

참고:
레퍼런스 이미지는 실제 브라우저 스크린샷이 아니라 최종 목표 분위기다.
특히 가져와야 할 것은:
1. 중앙 슬라임 중심성
2. 하늘/바닥 레이어 구분
3. 친구들의 반원형 생활감
4. 보라/골드 픽셀 UI 패널 톤
5. 소원 스트립과 뉴스 티커의 얇은 정보 배치
6. 상점 카드의 일관된 문법

그대로 따라하지 말고, 현재 게임 구조와 기능 해금 흐름에 맞게 번역해라.
초반 화면은 레퍼런스보다 30~40% 비어 있어야 하고, 레퍼런스 이미지는 late-game polished state 기준이다.
